Bölüm 1

Projeye Genel Bakış

Zemara, Türkçe dilli bir astroloji ve spiritüel rehberlik uygulamasıdır. Kullanıcılar doğum bilgilerini girerek kişiselleştirilmiş burç yorumları, doğum haritaları, tarot çekimleri, rüya yorumları, kristal rehberliği ve fal fotoğrafı analizi yapabilirler. Tüm analizler OpenAI GPT API ile gerçek zamanlı üretilmektedir.

🎯 Hedef Kitle

Türkiye'deki astroloji meraklıları, 18-35 yaş arası kadın ağırlıklı, mobil odaklı kullanıcılar

💡 Ana Değer Önerisi

Kişiselleştirilmiş AI yorumları — genel burç değil, kullanıcının kendi doğum verileriyle özelleştirilmiş içerik

📱 Platform

Web (Vercel) + Android APK/AAB (Capacitor) + iOS (yakında). Tek kod tabanı, tüm platformlar

💰 Gelir Modeli

Freemium: Ücretsiz / Premium ₺49/ay / VIP ₺99/ay. iyzico entegrasyonu ile Türk kartı desteği

SPA Navigasyon Sistemi

Uygulama tek HTML dosyasından oluşur. Her "sayfa" aslında <section id="page-xxx"> elementidir. showPage(id) fonksiyonu aktif section'ı gösterip diğerlerini gizler.

// app.js — showPage() özeti
function showPage(pageId) {
    document.querySelectorAll('.page').forEach(p => p.classList.remove('active'));
    document.getElementById(pageId)?.classList.add('active');
    // Sayfa bazlı veri yükleme
    if (pageId === 'moon') loadMoonCalendar();
    if (pageId === 'natal') ProfileMemory.fillAll();
    if (pageId === 'transit') initTransitForm();
    // ... diğer sayfalar
}
Bölüm 4

Ücretsiz Özellikler

1. Günlük Burç Yorumu

Kullanıcı adını ve doğum tarihini girer. Sistem burcu hesaplayıp GPT'ye gönderir. Kısa (200-300 kelime) günlük enerji yorumu üretilir. Sonuç 6 saat cache'lenir (aynı kullanıcı için).

  • Endpoint: POST /api/horoscope
  • Girdi: { name, birthDate, period: 'daily' }
  • Cache TTL: 6 saat (server-side Map)

2. Tek Kart Tarot

Rastgele 1 kart çekilir (Major + Minor Arcana dahil). Kartın yönü (dik/ters), anlamı ve günlük mesajı üretilir. Kart görsel olarak 3D flip animasyonuyla açılır.

  • Endpoint: POST /api/tarot
  • Girdi: { spreadType: 'single' }

3. Ay Takvimi

Seçilen ay için Yeni Ay ve Dolunay tarihlerini gösterir. Her evre için ritüel önerileri ve enerji açıklaması üretilir. Astronomik hesaplama lokal algoritmayla yapılır (API gerekmez).

Premium Özellikler ₺49/ay

4. Haftalık & Aylık Yorum

Günlük yorum ile aynı akış, period parametresi 'weekly' veya 'monthly' gönderilir. Daha uzun ve derinlemesine analiz (500-800 kelime).

5. Doğum Haritası Hesaplama

Doğum tarihi, saati ve şehri alınır. GPT Güneş, Ay, Yükselen (Ascendant), Merkür, Venüs, Mars, Jüpiter, Satürn, Uranüs, Neptün, Plüton'un hangi burçta ve evde olduğunu hesaplar. 12 ev sistemi (Placidus) kullanılır.

  • Endpoint: POST /api/natal ile type: 'chart'
  • Girdi: { birthDate, birthTime, birthCity }

6. Natal Yorum (Karakter Analizi)

Doğum haritasındaki gezegen pozisyonlarına göre kişilik analizi. Güneş burcu (dış kişilik), Ay burcu (iç dünya, duygular), Yükselen (ilk izlenim) üçlüsü detaylı yorumlanır.

7. 3 Kart Tarot (Geçmiş-Şimdi-Gelecek)

3 kart açılımı. Her kart pozisyonuna özel yorum üretilir. Kartlar sırayla 3D animasyonla açılır.

8. Rüya Yorumu

Kullanıcı rüyasını metin olarak yazar. GPT Jung psikolojisi, arketip teorisi ve sembolik analiz perspektifinden yorumlar. Günde 2 yorum hakkı (UsageLimiter ile kontrol edilir).

  • Endpoint: POST /api/dream
  • Girdi: { dream: "rüya metni" }

9. Kristal Rehberi

Kullanıcı şikayetini/ihtiyacını yazar. GPT 3 kristal önerir: isim, fotoğraf (emoji), nasıl kullanılacağı, hangi çakra ile ilişkili olduğu. Türkçe olarak detaylı rehberlik.

  • Endpoint: POST /api/crystal
  • Girdi: { concern: "kaygı, odak sorunu..." }

VIP Özellikler ₺99/ay

10. Transit Analizi

Seçilen tarihteki gezegen pozisyonlarının doğum haritasıyla ilişkisi analiz edilir. Örneğin Jüpiter şu an Boğa'da geziyorsa ve kullanıcı Boğa ise bu "Jüpiter dönemi" anlamına gelir. 3 aylık öngörü içerir.

  • Endpoint: POST /api/transit
  • Girdi: { birthDate, birthTime, currentDate, sunSign }

11. Uyum Analizi (İlişki Haritası)

İki kişinin doğum tarihleri alınır. Güneş uyumu (temel enerji), Venüs uyumu (aşk/çekim), Mars uyumu (çatışma/tutku) hesaplanır. Uyum skoru (0-100) ve ilişki önerileri üretilir.

  • Endpoint: POST /api/compatibility
  • Girdi: { person1: { name, birthDate }, person2: { name, birthDate } }

12. Celtic Cross Tarot (10 Kart)

En kapsamlı tarot açılımı. 10 kartın her biri farklı bir pozisyonu temsil eder: Mevcut durum, engel, geçmiş, gelecek, bilinçaltı, umutlar/korkular, dış etkiler, ipuçları, tahmin ve son sonuç. Pozisyonlara özel detaylı yorum üretilir.

13. Fal Fotoğrafı Yorumu

Kullanıcı el fotoğrafı, kahve fincanı veya başka bir fal fotoğrafı yükler. Görsel base64'e çevrilip OpenAI Vision API'a gönderilir. Fotoğraf türüne göre özel yorum üretilir (el falı, kahve falı, tarot kartı falı).

  • Endpoint: POST /api/fortune
  • Girdi: { image: "base64...", fortuneType: "palm|coffee|tarot", userName, birthDate }
  • Not: Görsel 5MB limitine tabi. Server'da 5mb body limit aktif.
Şu Anki Durum: Açık VIP Modu

isOpenVipAccessEnabled() fonksiyonu true döndürdüğünden tüm kullanıcılar VIP özelliklerine erişebilmekte. Bu, büyüme aşamasında kullanıcı kazanmak için bilerek seçilmiş bir stratejidir. Ödeme sistemi hazır olduğunda kapatılabilir.

AI Entegrasyonu

Model Seçimi: gpt-4o-mini

Tüm özellikler için gpt-4o-mini kullanılıyor. Seçim gerekçesi:

  • Maliyet: gpt-4o'nun ~5-10 katı daha ucuz
  • Hız: gpt-4o'dan ~3x daha hızlı yanıt
  • Yeterlilik: Burç/tarot yorumları için gpt-4o kalitesi gerekmez
  • Vision desteği: Fal fotoğrafı için görsel işleyebiliyor

Prompt Mühendisliği

Her özellik için sistem prompt'u ve kullanıcı prompt'u ayrı tasarlanmıştır. Genel kural:

// Sistem prompt yapısı (her özellik için benzer)
const systemPrompt = `Sen Zemara'in astroloji uzmanısın.
Türkçe yaz. Samimi, sıcak ve rehber bir ton kullan.
Yanıtı yapılandırılmış bölümlerle ver.
Kişisel veri: ${userName}, ${birthDate}, ${zodiacSign}`;

// Kullanıcı prompt'u — özelliğe özgü
const userPrompt = `${userName} için ${period} burç yorumu yaz...`;

Server-Side Response Cache

Aynı istek tekrar geldiğinde (aynı parametreler) OpenAI'ya gitmeyi önlemek için in-memory cache kullanılır:

// server.js — Cache sistemi
const responseCache = new Map(); // max 200 entry
const CACHE_TTL = {
    horoscope: 6 * 60 * 60 * 1000,    // 6 saat
    natal:     24 * 60 * 60 * 1000,    // 24 saat
    moon:      12 * 60 * 60 * 1000,    // 12 saat
    tarot:     0,                       // Cache yok (her çekim farklı)
    fortune:   0,                       // Cache yok (fotoğraf bazlı)
};

function getCacheKey(endpoint, params) {
    return endpoint + ':' + JSON.stringify(params);
}

Token Optimizasyonu

Her özellik için prompt'lar optimizasyona tabi tutulmuştur:

  • Gereksiz açıklamalar kaldırıldı (sistem prompt kısaltıldı)
  • max_tokens özelliğe göre ayarlandı (günlük: 600, natal: 1200, celtic cross: 2000)
  • temperature: 0.8 — yaratıcı ama tutarlı yanıtlar için
Bölüm 6

Authentication Sistemi

Mimari: localStorage-Tabanlı

Sunucuya kimlik doğrulaması için istek gönderilmez. Session tamamen tarayıcı localStorage'ında saklanır. Bu yaklaşım basitlik ve çevrimdışı çalışma sağlar, ancak server-side doğrulama yoktur.

// AuthSystem._key = 'zemara_session'
// Session yapısı:
{
    id: 'user_abc123',
    name: 'Ayşe Kaya',
    email: 'ayse@ornek.com',
    birthDate: '1990-05-15',
    birthCity: 'İstanbul',
    plan: 'vip',          // 'free' | 'premium' | 'vip'
    loginAt: 1710000000000,
    isDemo: false
}

Şifre Hashleme

// SHA-256 + statik salt
async function hashPassword(password) {
    const saltedPassword = password + 'zemara_v4_salt_';
    const msgBuffer = new TextEncoder().encode(saltedPassword);
    const hashBuffer = await crypto.subtle.digest('SHA-256', msgBuffer);
    return Array.from(new Uint8Array(hashBuffer))
                .map(b => b.toString(16).padStart(2, '0')).join('');
}
Güvenlik Notu

Statik salt ideal değildir; her kullanıcı için benzersiz salt kullanılmalıdır. Ancak mevcut mimari (localStorage-only) için yeterlidir. Server-side DB auth'a geçişte bcrypt + benzersiz salt kullanılmalı.

ProfileMemory (Otomatik Doldurma)

Kullanıcı bir formu doldururken sistem verileri localStorage'a kaydeder. Bir sonraki sayfada ProfileMemory.fillAll() çağrılarak tüm formlar otomatik doldurulur. Bu, kullanıcı deneyimini önemli ölçüde iyileştirir.

// ProfileMemory.save() — herhangi bir sayfadan çağrılabilir
ProfileMemory.save({
    name: 'Ayşe',
    birthDate: '1990-05-15',
    birthCity: 'İstanbul',
    sunSign: 'Boğa'
});

// ProfileMemory.fillAll() — sayfa yüklenince form alanlarını doldurur
// data-profile-field="birthDate" → input value = '1990-05-15'

Demo Onboard (Kişiselleştirilmiş Demo)

Demo butonuna tıklandığında kullanıcıdan isim, doğum tarihi ve şehir alınır. Bu veriler session'a ve ProfileMemory'e kaydedilir. Tüm AI özellikleri o kişinin verileriyle çalışır.

Teknik Mimari: i18n Katmanı

Mevcut sistem tamamen Türkçe hardcoded'dır. i18n için iki katmanda değişiklik gereklidir:

1. UI Metinleri (app.js translations objesi)

Uygulama içinde zaten kısmi bir translations objesi mevcut. Bu genişletilecek:

// app.js — mevcut yapı (genişletilecek)
const translations = {
    tr: {
        nav_daily: 'Günlük',
        nav_natal: 'Doğum Haritası',
        nav_tarot: 'Tarot',
        btn_start: 'Ücretsiz Başla',
        demo_btn: '⚡ Demo Girişi — Tüm Özellikler Açık',
        // ... ~200 string
    },
    en: {
        nav_daily: 'Daily',
        nav_natal: 'Birth Chart',
        nav_tarot: 'Tarot',
        btn_start: 'Start Free',
        demo_btn: '⚡ Demo Access — All Features Unlocked',
        // ... ~200 string
    },
    de: {
        nav_daily: 'Täglich',
        nav_natal: 'Geburtshoroskop',
        nav_tarot: 'Tarot',
        btn_start: 'Kostenlos starten',
        // ...
    },
    // fr, es, it, pt, pl, nl, ro...

function t(key) {
    const lang = getCurrentLang(); // localStorage veya tarayıcı dili
    return translations[lang]?.[key] ?? translations['en']?.[key] ?? key;
}

3. Dil Algılama ve Seçimi

// app.js — otomatik dil algılama
function detectLanguage() {
    // 1. Kullanıcının manuel seçimi (en yüksek öncelik)
    const saved = localStorage.getItem('zemara_lang');
    if (saved) return saved;

    // 2. Tarayıcı dili
    const browserLang = navigator.language?.split('-')[0]; // 'de-DE' → 'de'
    const supported = ['tr','en','de','fr','es','it','pt','pl','nl','el','ro','hu','cs','sv','da','fi','no'];
    if (supported.includes(browserLang)) return browserLang;

    // 3. Fallback: İngilizce
    return 'en';
}

function setLanguage(lang) {
    localStorage.setItem('zemara_lang', lang);
    applyTranslations(lang);
    // Sayfa yeniden render'lamaya gerek yok — DOM'daki data-i18n attr'ları güncellenir
}

4. HTML Attr Tabanlı Çeviri Sistemi

<!-- index.html — data-i18n attr ekle -->
<button data-i18n="btn_start" class="btn-primary">Ücretsiz Başla</button>
<span data-i18n="nav_daily">Günlük</span>

<!-- app.js — applyTranslations() -->
function applyTranslations(lang) {
    document.querySelectorAll('[data-i18n]').forEach(el => {
        const key = el.getAttribute('data-i18n');
        el.textContent = t(key);
    });
    // placeholder'lar için:
    document.querySelectorAll('[data-i18n-placeholder]').forEach(el => {
        el.placeholder = t(el.getAttribute('data-i18n-placeholder'));
    });
}
